# JavaScript Problem Solving & Algorithms 🚀

Este repositorio contiene una colección de desafíos de algoritmos y estructuras de datos resueltos en JavaScript (ES6+). El objetivo es fortalecer el pensamiento lógico y prepararme para pruebas técnicas de alto nivel.

## 📌 Objetivos de este Repositorio
- Implementar soluciones eficientes utilizando patrones de diseño de algoritmos.
- Analizar la complejidad de cada solución mediante la **Notación Big O**.
- Mantener un código limpio, legible y documentado.

## 🛠️ Tecnologías y Herramientas
- **Lenguaje:** JavaScript (Node.js)

## 📂 Contenido del Repositorio

Los ejercicios están organizados por categorías o patrones para facilitar el estudio:

### 1. Patrones de Frecuencia (Frequency Counters)
*Utilizados para evitar bucles anidados $O(n^2)$ y mejorar el rendimiento a $O(n)$.*
- [x] **Same Frequency:** Comprobar si dos números tienen los mismos dígitos.
- [ ] **Anagrams:** Validar si una palabra es un anagrama de otra.

### 2. Múltiples Punteros (Multiple Pointers)
*Eficientes para buscar pares de valores en arreglos ordenados con mínima memoria.*
- [x] **Sum Zero:** Encontrar el primer par cuya suma sea 0.
- [ ] **Unique Values:** Contar valores únicos en un arreglo.

### 3. Ventana Deslizante (Sliding Window)
*Ideal para subconjuntos de datos en arreglos o cadenas.*
- [ ] **Max Subarray Sum:** Calcular la suma máxima de 'n' elementos consecutivos.

### 4. Lógica Fundamental y Aritmética
*Ejercicios clásicos para fortalecer el control de flujo y operadores básicos.*
- [x] **FizzBuzz:** Manejo de condicionales y operador de módulo `%`.

## 📈 Análisis de Complejidad (Big O)

Para cada ejercicio, incluyo un comentario con su respectivo análisis de eficiencia:

- **Complejidad de Tiempo:** ¿Cuánto tarda en ejecutarse según crecen los datos?
- **Complejidad de Espacio:** ¿Cuánta memoria adicional requiere?

## 🚀 Cómo usar este repositorio

1. Clona el proyecto:
   `git clone https://github.com/leysanchezb92/algoritmos-js.git`
2. Navega a la carpeta del ejercicio:
   `cd ejercicios/patron-frecuencia`
3. Ejecuta el archivo con Node:
